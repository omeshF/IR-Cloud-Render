Key differences for incident response

Vercel — Serverless

  • Logs are ephemeral — lost on cold start
  • No persistent state between requests
  • Each function runs in isolation
  • IR must collect logs immediately
  • Cannot track login attempts across calls
  • Harder to correlate events over time

Render — Persistent server

  • Logs survive for the life of the process
  • In-memory state accumulates over time
  • Session store grows until restart
  • Full attack history available
  • Can track brute force across attempts
  • Richer timeline reconstruction possible

Unique to Render — persistent endpoints

These endpoints expose data that only exists because the server has been running continuously. On Vercel they would return empty results on every cold start.
